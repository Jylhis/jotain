* Something, from scratch

Jotain is Finnish for “something” — a custom GNU Emacs configuration with no framework underneath. Instead of layering on Doom or Spacemacs, it is built from scratch: Nix builds the editor from source, plain Elisp configures it.

[ quickstart ] [ view on github ] gnu emacs 30+ · nix · elisp

** DONE What ships

- emacs.nix — Emacs from source, every build flag exposed, binary-cache parity with nixpkgs
- lisp/init-*.el — modular config, one file per concern; magit lives with vc, dirvish with dired
- default.nix — distribution wrapper + all ~275 tree-sitter grammars
- module.nix — home-manager service: daemon + emacsclient
- devenv.nix — the same Emacs for developers and tools
- Justfile — check, build, run, pins, cleanup
// currently
› targeting emacs 30.1 — native comp, tree-sitter, lexical binding
› setopt for every defcustom, so :set callbacks actually run

** Quickstart

#+begin_src bash
git clone https://github.com/Jylhis/jotain.git && cd jotain
devenv shell   # emacs 30.1, just, nil, treefmt, nerd fonts
just run       # GUI — --init-directory=<repo>, never touches ~/.emacs.d
just build     # standalone: emacs 30 + 275 tree-sitter grammars
#+end_src

First launch bootstraps anything Nix doesn’t provide from MELPA; later launches load from the package cache.

KEYBINDINGS(7)

Keybindings

Jotain follows Emacs conventions: only global rebindings live in lisp/init-keys.el; per-package bindings stay colocated with the package’s use-package block. which-key is enabled by default — after any prefix, a paged menu of continuations appears in the echo area.

C-c — user namespace

Embark habits

// search → edit
consult-ripgrep → C-. E exports to a grep buffer, then C-x C-q (wgrep) makes it editable — rewrite every match in place and save across all files at once.

Disabled by default

C-z and C-x C-z — accidentally suspending Emacs from a GUI frame is never what you want.
