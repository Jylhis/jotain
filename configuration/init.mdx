# init.el

The `init.el` file is Jotain's main configuration entry point. It is loaded after `early-init.el` and handles package setup, theme loading, base settings, and module loading.

## Structure

### Package Archives

MELPA is added as a package archive alongside the default GNU ELPA:

```elisp
(add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/") t)
(package-initialize)
```

### Load Path

Custom modules from `lisp/` are added to the load path:

```elisp
(add-to-list 'load-path (expand-file-name "lisp" user-emacs-directory))
```

### Custom File

Emacs automatic customizations are stored separately to keep `init.el` clean:

```elisp
(setq custom-file (locate-user-emacs-file "custom.el"))
(when (file-exists-p custom-file) (load custom-file))
```

### Theme

Jotain uses the built-in Modus Vivendi Tritanopia theme:

```elisp
(load-theme 'modus-vivendi-tritanopia t)
```

### Base Settings

Core Emacs behavior is configured via `use-package emacs`:

- `fill-column` set to 100
- Short answers enabled (`y/n` instead of `yes/no`)
- Dialog boxes disabled
- Delete moves to trash
- `C-z` and `C-x C-z` unbound (no accidental suspend)
- `M-o` bound to `other-window`

### Packages

External packages are loaded with `use-package`:

- **nix-ts-mode** — Tree-sitter mode for Nix files
- **markdown-mode** — Markdown editing with GFM mode for README files
