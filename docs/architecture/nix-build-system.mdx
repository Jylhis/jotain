## Flake structure

The flake uses `flake-parts` for modular Nix configuration and `nix-community/emacs-overlay` for bleeding-edge Emacs builds and packages.

```
flake.nix                  # Entry point
├── emacs.nix              # Emacs build definition
├── default.nix            # Jotain package
├── shell.nix              # Dev shell
├── nix/overlays/default.nix  # Custom overlay
├── nix/modules/home/      # Home Manager module
└── nix/modules/nixos/     # NixOS module
```

## Key files

### `emacs.nix`

Builds Emacs 30 (PGTK variant) with:
- All Emacs packages auto-extracted from `use-package` declarations
- The `jotain-modules` package (compiled elisp)
- Runtime dependencies (LSP servers, fonts, CLI tools)
- Tree-sitter grammars with `TREE_SITTER_DIR` injected

### `default.nix`

Defines the `jotain` package — copies `init.el` and `early-init.el` to `share/jotain/` for the Home Manager module to link into place.

### `nix/overlays/default.nix`

The overlay provides:
- **`jotain-modules`**: Compiles `elisp/` into an Emacs package via `trivialBuild`
- **Custom packages**: Packages not in nixpkgs (e.g., `claude-code-ide`) are defined here as `trivialBuild` derivations

### `shell.nix`

Development shell with `emacs-dev` and `jot` wrappers, LSP servers, formatters, and other dev tools.

## Flake outputs

| Output | Description |
|--------|-------------|
| `packages.default` | The full Jotain Emacs build |
| `homeModules.default` | Home Manager integration module |
| `nixosModules.default` | NixOS module |
| `devShells.default` | Development environment |
| `checks.*` | Test suite (ERT, NMT, formatting) |
