---
title: Home Manager Module
description: Configure Jotain through Home Manager options
---

Jotain exposes a Home Manager module at `homeModules.default` with the following options.

## Options

<ResponseField name="programs.jotain.enable" type="boolean" default="false">
  Master switch to enable the Jotain Emacs configuration. When enabled, Jotain installs Emacs with all configured packages and sets up configuration files.
</ResponseField>

<ResponseField name="programs.jotain.package" type="package" default="pkgs.jotain">
  The Jotain configuration package to use. Override this if you have a custom build.
</ResponseField>

<ResponseField name="programs.jotain.enableDaemon" type="boolean" default="true">
  Run Emacs as a systemd service (Linux) or launchd service (macOS). This keeps Emacs running in the background for instant client connections.
</ResponseField>

<ResponseField name="programs.jotain.includeRuntimeDeps" type="boolean" default="true">
  Install runtime dependencies: LSP servers, fonts, CLI tools, and tree-sitter grammars. Disable this if you manage these dependencies separately.
</ResponseField>

<ResponseField name="programs.jotain.extraPackages" type="function" default="epkgs: []">
  Add extra Emacs packages beyond what Jotain includes by default.
</ResponseField>

## Examples

### Minimal configuration

```nix
programs.jotain.enable = true;
```

### Add extra packages

```nix
programs.jotain = {
  enable = true;
  extraPackages = epkgs: [
    epkgs.pdf-tools
    epkgs.org-roam
    epkgs.elfeed
  ];
};
```

### Disable daemon mode

```nix
programs.jotain = {
  enable = true;
  enableDaemon = false;
};
```

### Manage runtime deps separately

```nix
programs.jotain = {
  enable = true;
  includeRuntimeDeps = false;
};

# Install your own LSP servers, fonts, etc.
home.packages = with pkgs; [
  nil  # Nix LSP
  gopls
];
```
