# early-init.el

The `early-init.el` file is loaded by Emacs 30+ before package initialization and GUI setup. It handles settings that must be applied as early as possible.

## What It Does

### Load Preference

In non-interactive sessions, source files are preferred over byte-compiled files to prevent stale bytecode:

```elisp
(setopt load-prefer-newer noninteractive)
```

### use-package imenu

Enables imenu support for `use-package` declarations, making it easy to navigate configuration blocks:

```elisp
(setopt use-package-enable-imenu-support t)
```

### GUI Settings

Applied before the GUI renders to prevent visual flicker:

```elisp
(tool-bar-mode -1)
(setopt inhibit-startup-screen t)
```
