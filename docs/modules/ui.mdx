The `ui` module (`elisp/ui.el`) manages themes, visual polish, and the startup dashboard.

## Theme system

Jotain uses a Nord-based dual theme system:

| Theme variable | Maps to | Style |
|---------------|---------|-------|
| `jotain-theme-light` | `doom-nord-light` | Light |
| `jotain-theme-dark` | `nord` | Dark |

**Toggle**: Press `C-c t` to switch between light and dark themes.

### Theme blending prevention

Before loading any theme, `jotain-ui--disable-all-themes` is called automatically to prevent visual artifacts from theme stacking. This is implemented as advice on `load-theme`.

### Daemon mode

In daemon mode, themes are applied after the first frame is created via `server-after-make-frame-hook` to ensure colors render correctly.

## Dashboard

The startup screen uses [Enlight](https://github.com/ichernyshovvv/enlight) to display a clean dashboard with quick-access actions.

## Visual packages

- **doom-themes** — Theme collection (Nord variants)
- **doom-modeline** — Modern, informative mode line
- **nerd-icons** — Icon support throughout the UI
- **diff-hl** — Highlight version control changes in the fringe
