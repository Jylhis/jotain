The `programming` module (`elisp/programming.el`) provides a full IDE-like development experience.

## LSP via Eglot

Jotain uses Emacs's built-in [Eglot](https://www.gnu.org/software/emacs/manual/html_mono/eglot.html) as the LSP client. Language servers are provided by Nix and available on PATH.

### Included LSP servers

| Server | Language |
|--------|----------|
| `nil` | Nix |
| `gopls` | Go |
| `bash-language-server` | Bash |
| `typescript-language-server` | TypeScript / JavaScript |
| `yaml-language-server` | YAML |
| `marksman` | Markdown |
| `clang-tools` (clangd) | C / C++ |

### LSP keybindings

| Key | Action |
|-----|--------|
| `M-.` | Go to definition |
| `M-,` | Go back |
| `M-?` | Find references |
| `C-c l r` | Rename symbol |
| `C-c l a` | Code action |

## Tree-sitter

Tree-sitter grammars are bundled via Nix for 17 languages: Bash, C, C++, CSS, Go, Go modules, HTML, JavaScript, JSON, Lua, Markdown, Nix, Python, Rust, TSX, TypeScript, and YAML.

Grammar paths are set in `early-init.el` from the `TREE_SITTER_DIR` environment variable injected at build time.

`treesit-auto` is configured to automatically use tree-sitter major modes when grammars are available.

## DAP debugging

Debug Adapter Protocol support is available via `dape` for interactive debugging sessions.

## Language-specific modes

Language modes for Go, Rust, Nix, TypeScript, Markdown, YAML, and more are included and configured with appropriate tree-sitter integration.
