The `completion` module (`elisp/completion.el`) sets up a modern completion stack.

## Components

| Package | Role |
|---------|------|
| **Vertico** | Vertical minibuffer completion UI |
| **Corfu** | In-buffer completion popup |
| **Consult** | Enhanced search and navigation commands |
| **Orderless** | Flexible completion matching (space-separated patterns) |
| **Marginalia** | Rich annotations in the minibuffer |
| **Embark** | Context actions on completion candidates |
| **Cape** | Additional completion-at-point backends |

## Key commands

| Key | Command | Description |
|-----|---------|-------------|
| `C-x b` | `consult-buffer` | Switch buffer with preview |
| `C-s` | `consult-line` | Search in current buffer |
| `M-s r` | `consult-ripgrep` | Search across project files |
| `M-s f` | `consult-fd` | Find files by name |
| `C-.` | `embark-act` | Context action on current candidate |
| `M-g g` | `consult-goto-line` | Go to line number |

## Completion styles

Orderless allows flexible matching with space-separated patterns:

- `foo bar` matches candidates containing both "foo" and "bar" in any order
- Supports regexp, initialism, and literal matching styles
