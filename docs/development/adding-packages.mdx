## Emacs packages

### Auto-extracted packages

Most packages are added with a single step — add a `use-package` form to the appropriate `elisp/*.el` module:

```elisp
(use-package rainbow-delimiters
  :hook (prog-mode . rainbow-delimiters-mode))
```

If the package name matches its nixpkgs attribute, it's automatically extracted and installed. No Nix changes needed.

### Name mismatches

When the Elisp name differs from the nixpkgs attribute, add a mapping in `nix/lib/dependencies.nix`:

```nix
packageNameMap = {
  "some-elisp-name" = "actual-nixpkgs-name";
};
```

### Packages not in nixpkgs

For packages not available in nixpkgs, add a `trivialBuild` in `nix/overlays/default.nix`:

```nix
my-custom-package = pkgs.emacsPackages.trivialBuild {
  pname = "my-custom-package";
  version = "0.1.0";
  src = pkgs.fetchFromGitHub {
    owner = "...";
    repo = "...";
    rev = "...";
    hash = "...";
  };
};
```

## Runtime tools

For LSP servers, CLI tools, fonts, and tree-sitter grammars, add them to `nix/lib/runtime-deps.nix`:

```nix
# LSP server
lspServers = [ pkgs.rust-analyzer ];

# CLI tool
cliTools = [ pkgs.jq ];

# Font
fonts = [ pkgs.fira-code ];

# Tree-sitter grammar
treesitterGrammars = [ pkgs.tree-sitter-grammars.tree-sitter-rust ];
```

## Checklist

<Steps>
  <Step title="Add use-package form">
    Add the `use-package` configuration in the appropriate `elisp/*.el` module.
  </Step>
  <Step title="Handle name mismatches">
    If the package name doesn't match nixpkgs, add a mapping in `dependencies.nix`.
  </Step>
  <Step title="Custom builds if needed">
    For packages not in nixpkgs, add a `trivialBuild` in the overlay.
  </Step>
  <Step title="Add runtime deps">
    For LSP servers, CLI tools, fonts, or grammars, add to `runtime-deps.nix`.
  </Step>
  <Step title="Test">
    Run `just test` to verify everything builds and tests pass.
  </Step>
</Steps>
