## Test architecture

Jotain has three layers of testing:

1. **ERT tests** (`tests/`) — Elisp unit and integration tests
2. **NMT tests** (`nmt-tests/`) — Nix Module Tests for Home Manager
3. **VM runtime test** — Full NixOS VM validation (CI only)

## ERT tests

Tests live in `tests/` and are auto-discovered by `tests/test-all.el`, which loads all `test-*.el` files (excluding `test-all.el`, `test-helpers.el`, and `test-suite-*.el`).

### Running tests

```bash
just test-smoke   # Smoke tests (< 1s)
just test-fast    # Fast unit tests (< 5s)
just test         # Full suite
just test-tag unit  # By tag
```

### Writing tests

Tests use ERT (Emacs Lisp Regression Testing) and are tagged for selective execution:

```elisp
(ert-deftest test-my-feature ()
  :tags '(unit fast)
  (should (equal (my-function 1 2) 3)))
```

### Test isolation

- Tests run with `HOME=.dev-home` to avoid contaminating user config
- `test-helpers.el` mirrors `early-init.el` settings since tests run with `-Q` (no user init)

## NMT tests

NMT (Nix Module Tests) validate the Home Manager module by building actual configurations and checking outputs.

### Available tests

| Test | Validates |
|------|-----------|
| `test-module-enabled` | Config files, directories, aliases, systemd service, fonts |
| `test-module-disabled` | No config created when module is disabled |

### Running NMT tests

```bash
just test-nmt

# Or a specific test:
nix build .#checks.x86_64-linux.test-module-enabled --print-build-logs
```

### Adding NMT tests

Add a new entry in `nmt-tests/default.nix`:

```nix
test-my-feature = mkTest "my-feature" ''
  set -euo pipefail
  homeConfig="${buildHomeConfig [
    { programs.emacs = { enable = true; userConfig = ./..; }; }
  ]}"
  # Assertions here
  touch $out
'';
```

## CI

GitHub Actions runs on push to `main` and on pull requests:

1. Builds the default package
2. Runs `nix flake check` (all ERT tests, NMT tests, formatting)
3. Uses [Cachix](https://cachix.org/) (`jylhis` + `nix-community`) for binary caching
