title: Commands Reference
description: All available development commands via just
---

All commands are run via `just` (see `justfile`). Enter the dev shell with `nix develop` first.

## Building

| Command | Description |
|---------|-------------|
| `just build` | Build Emacs package with Nix |
| `just compile` | Byte-compile all `.el` files directly (no Nix) |
| `just check` | Syntax check via Nix dry-run |

## Testing

| Command | Description | Speed |
|---------|-------------|-------|
| `just test-smoke` | Ultra-fast smoke tests | < 1s |
| `just test-fast` | Fast unit tests | < 5s |
| `just test` | Full ERT test suite via Nix | Minutes |
| `just test-tag TAG` | Run ERT tests by tag | Varies |
| `just test-nmt` | NMT Home Manager integration tests | Minutes |
| `just test-all` | All checks except VM runtime | Minutes |
| `just test-all-plus-runtime` | Includes VM nixosTest (CI only) | Slow |
| `just test-runtime` | VM-based runtime validation | Slow |

### Test tags

| Tag | Purpose |
|-----|---------|
| `smoke` | Critical path validation |
| `fast` | Quick unit tests |
| `unit` | Standard unit tests |
| `integration` | Cross-module integration |
| `slow` | Long-running tests |

## Checking

| Command | Description | Speed |
|---------|-------------|-------|
| `just check-instant` | Formatting + smoke only | < 10s |
| `just check-fast` | Above + fast unit tests | < 1 min |
| `just check-parallel` | All checks in parallel | Faster on multi-core |

## Formatting

| Command | Description |
|---------|-------------|
| `just format` | Format all files via treefmt-nix |
| `nix fmt` | Same as above |

## Nix

| Command | Description |
|---------|-------------|
| `nix flake update` | Update all flake inputs |
| `nix flake update INPUT` | Update a specific input |
| `just info-nix` | Show flake outputs |
| `just info-checks` | Describe available checks |

## Running a specific NMT check

```bash
nix build .#checks.x86_64-linux.test-module-enabled --print-build-logs
```
