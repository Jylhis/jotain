# Quickstart

## Clone the repository

```bash
git clone https://github.com/Jylhis/jotain.git
cd jotain
```

## Enter the development shell

Jotain ships a [devenv](https://devenv.sh) shell that provides `just`, the Nix tooling (`nil`, `nixfmt-rfc-style`, `statix`, `deadnix`, `treefmt`), Meson/Ninja, `sonarlint-ls`, `rass`, `eca`, `tagref`, `docker-langserver`, the docs chain, and the Nerd Fonts the UI config looks up. With `direnv` installed it activates automatically; otherwise enter it manually:

```bash
devenv shell
```

<Note>
Emacs is **temporarily not installed into the devenv shell** — its ~1 GB closure dominated `direnv allow` time (see the top-of-file note in `devenv.nix`). The `just run` / `debug` / `tty` / `daemon` / `client` / `quick` / `compile` / `test` / `bench` recipes print a disabled notice and exit until Emacs is wired back in. Use `just run-built` (below) to build Emacs via Nix and launch it.
</Note>

## Run Emacs with the Jotain configuration

```bash
just run-built      # build Emacs via Nix for this platform, then launch it
```

`just run-built` launches the Nix-built Emacs with `--init-directory=<repo>`, so it never touches `~/.emacs.d`. The first launch bootstraps any package not provided by Nix from MELPA; subsequent launches load from the package cache.

When the `just run` / `daemon` / `client` recipes are re-enabled, see [Launching Emacs](/usage/launching) for when to reach for each pattern (one-shot vs. daemon + client vs. `-Q -nw` quick edit) and how to enable the `emd` / `em` / `emg` shell aliases via `services.jotain.shellAliases`.

## Build a standalone Emacs

If you want a Nix-built binary outside the devenv shell:

```bash
just build          # mainline Emacs 31 (emacs-git master) + all ~275 tree-sitter grammars
just run-built      # build for the current platform, then launch result/bin/emacs
```

## Sanity-check the configuration

```bash
just check          # nix flake check: treefmt, statix, deadnix, elisp paren-check + byte-compile
just fmt            # nix fmt — Nix formatting via the shared treefmt config
```

(`just compile` and `just test` are disabled stubs for now; their byte-compile coverage runs inside `just check` via the `elisp-compile` flake check.)

## Clean build artifacts

```bash
just clean          # .elc files, autosaves, eln-cache, result/
just clean-all      # also elpa/, var/, etc/, .dev-home/ — forces a full re-fetch
```

## Update pinned sources

`flake.lock` is the single source of truth for input revs; `devenv.yaml` mirrors the shared ones (`nixpkgs`, `treefmt-nix`).

```bash
just update             # update every flake input, then sync devenv.yaml + devenv.lock
just verify             # assert flake.lock and devenv.lock agree on every shared input
```
