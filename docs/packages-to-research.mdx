# Packages worth researching

The list below is parked work, not rejected work. Each entry has a
short description of what the package does, why we left it out, and
what to evaluate before flipping it on.

## combobulate

**What it is**

A structural editor for tree-sitter languages by Mickey Petersen. It
turns the parse tree into the unit of motion and editing — instead of
moving the cursor by character or line you move by sibling, parent,
or named-child node. Once you internalise it, refactoring TypeScript,
Python, JSX, YAML, etc. starts to feel like editing lisp with paredit.

What it gives you:

- **Structural navigation.** `next-sibling`, `previous-sibling`,
  `into-parent`, `into-child` — all aware of the AST. No more
  fighting whitespace to "move to the next argument".
- **Structural selection.** `combobulate-mark-node-dwim` grows the
  selection by node type instead of character.
- **Structural edits.** Swap sibling nodes (reorder function
  arguments, array elements, JSX props); splice a node out; envelope
  a node in a wrapping construct (try/catch, if/else, for loop).
- **Avy-style jump** to any visible node.
- **Per-language envelopes** — a small DSL for "wrap this node in
  THIS template", which gives you `surround-with-…` operations for
  free.
- Built-in support for `python-ts-mode`, `typescript-ts-mode`,
  `tsx-ts-mode`, `js-ts-mode`, `json-ts-mode`, `yaml-ts-mode`,
  `html-ts-mode`, `css-ts-mode`, `go-ts-mode`, `rust-ts-mode`,
  `c-ts-mode`, `c++-ts-mode`.

**Why parked**

- Steep learning curve. It has its own mental model and ~20
  keybindings under one prefix; "drop it in and discover it
  organically" doesn't work.
- README still labels it work-in-progress. Some operations are
  rough on the edges depending on the grammar.
- Mickey Petersen's [introduction post][cb-intro] is the best
  starting point and is required reading before adopting it. Until
  someone has time to read it carefully, dropping it into the config
  would be cargo culting.

**What to evaluate when revisiting**

- Whether jotain has enough actual treesit-mode usage to justify it
  (i.e. are we writing TypeScript / Go / Python here, or just
  reading other people's code).
- Whether `treesit-fold` (already enabled) and the built-in treesit
  navigation cover the same ground for our usage.
- Read the intro post end to end and pick a small subset of bindings
  to learn first.

**Links**

- Source: <https://github.com/mickeynp/combobulate>
- Intro: <https://www.masteringemacs.org/article/combobulate-structured-editing-and-movement-emacs>

## gited

**What it is**

A Dired-style buffer for managing git branches. You get the
familiar `RET` / `g` / `d` / `m` / `u` keybindings but the rows are
branches instead of files: rename, delete, merge, push, fetch,
checkout, all without leaving the buffer.

**Why parked**

- Magit already covers branch management via `b` (`magit-branch`)
  in the status buffer with arguably better ergonomics for
  most flows.
- The Dired idiom is appealing if you have *many* branches and
  want to bulk-operate on them, but jotain doesn't have that
  scale right now.
- One more package to track for a feature that overlaps with
  magit.

**What to evaluate when revisiting**

- Whether you find yourself with 20+ branches in active
  repositories (gited shines at scale).
- Whether you want bulk rename / delete / merge across many
  branches.
- Whether magit's branch popup is feeling slow for the use case.
