## Emacs won't start

Run Emacs with debug output to identify the error:

```bash
emacs --debug-init
```

Check the `*Messages*` and `*Warnings*` buffers for details.

## LSP not working

Verify the language server is on PATH:

```
M-x shell-command RET which nil RET
```

If the server isn't found, ensure `includeRuntimeDeps = true` in your Home Manager config:

```nix
programs.jotain = {
  enable = true;
  includeRuntimeDeps = true;  # default
};
```

## Fonts look wrong

Check which fonts Jotain detected:

```
C-c f i
```

Ensure runtime dependencies are enabled for bundled fonts. If using Home Manager, also verify:

```nix
fonts.fontconfig.enable = true;
```

## Theme blending artifacts

This should not happen with Jotain's advised `load-theme`. If it does:

1. Run `M-x jotain-ui--disable-all-themes`
2. Toggle the theme with `C-c t`

## Daemon not starting

Check the service status and logs:

```bash
systemctl --user status emacs.service
journalctl --user -u emacs.service
```

## Configuration not applying

After changing your Nix configuration, rebuild and restart the daemon:

```bash
# Rebuild
home-manager switch --flake .

# Restart daemon
systemctl --user restart emacs.service
```

## Tree-sitter not highlighting

Verify tree-sitter grammars are available:

```
M-x treesit-language-available-p RET nix RET
```

If not available, check that `TREE_SITTER_DIR` is set correctly in your environment. This is normally handled automatically by `emacs.nix`.
