# 2026-07-01

## Switch prose font hook from `variable-pitch-mode` to `mixed-pitch-mode` (keep Org tables and code aligned)

### Summary

`init-writing.el` applied `(text-mode . variable-pitch-mode)` to every
`text-mode`-derived buffer. `variable-pitch-mode` makes the *whole* buffer
proportional, so Org (which aligns tables by padding cells with spaces) rendered
ragged tables: a monospace-only trick applied under a proportional font. The
same hit source blocks and verbatim spans.

Swapped in the `mixed-pitch` package (MELPA, jabranham). `mixed-pitch-mode`
remaps the buffer default to `variable-pitch` for prose while keeping a curated
face list (`mixed-pitch-fixed-pitch-faces` — `org-table`, `org-code`,
`org-block`, `org-verbatim`, `org-formula`, …) pinned to `fixed-pitch`. Org
tables align again; prose stays proportional. It reuses the same
`variable-pitch` face that `init-ui.el` already configures, so no font/theme
change was needed.

Two follow-on decisions:
- **Commit-message and email buffers are now fully monospaced.** They derive
  from `text-mode` and were proportional before; they are column-sensitive
  (50/72 rule), so a new opt-out turns `mixed-pitch-mode` back off there.
- **Markdown uses the `mixed-pitch` default face list** (no extra faces added);
  Org is the case that mattered and is covered out of the box.

### Changes

- **`lisp/init-writing.el`**
  - Dropped `(text-mode . variable-pitch-mode)` from the `text-mode`
    `use-package` `:hook` (kept `visual-line-mode`, `visual-wrap-prefix-mode`)
    and reworded its `;;; @doc`.
  - Added a `use-package mixed-pitch` block. `:hook` enables `mixed-pitch-mode`
    in `text-mode`, and a `:preface` helper `jotain-writing--keep-monospace`
    (calls `(mixed-pitch-mode -1)`) is hooked to `git-commit-setup` and
    `message-mode` for the monospace opt-out. `git-commit-setup-hook` and
    `message-mode-hook` both run after `text-mode-hook`, so the opt-out cleanly
    reverses it. Added `(declare-function mixed-pitch-mode "mixed-pitch" …)`.
  - The helper lives in `:preface` (not between the `;;; @doc` and the
    `use-package` form) so `nix/packages-doc.nix` still associates the doc block
    with the block — otherwise the generated reference showed `_Undocumented._`.

- **`lisp/init-lang-data.el`** — in `jotain-lang-data--enable-prog-mode-features`
  (the YAML prose-off workaround), changed `(variable-pitch-mode -1)` to
  `(mixed-pitch-mode -1)` and added the matching `declare-function`; without
  this YAML buffers would keep a proportional font. Updated the docstring
  wording.

- **`bench/find-file-bench.el`** — updated the timed-minor-modes list
  (`variable-pitch-mode` → `mixed-pitch-mode`) so the find-file benchmark
  measures the mode that now fires in `text-mode`.

- **`docs/configuration/package-reference.mdx`** — regenerated via
  `just docs-refresh-packages` from the updated `;;; @doc` blocks (adds a
  `### mixed-pitch` section, reworks `### text-mode`). Not hand-edited; CI's
  `packages-doc-in-sync` check verifies the in-tree copy.

Nix build needed no manual package entry: `nix/use-package.nix` auto-scans the
new `(use-package mixed-pitch …)` and resolves it to `epkgs.mixed-pitch`.

### Verification

  present, no `_Undocumented._`.
- `just check` — flake check (byte-compile with warnings-as-errors, paren

- Manual (`just run-built`): Org numeric table aligns and right-justifies; YAML

  monospaced.

### Compatibility note

Only `text-mode`-derived buffers change. `prog-mode` and code modes were never
hooked and are untouched; `conf-mode` is not `text-mode`-derived. `denote`
notes and Markdown prose stay proportional (now with monospaced code spans).
`init-ui.el` faces and `init-org.el`/org-modern are unchanged.
