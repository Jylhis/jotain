# 2026-05-19

## YAML buffers still rendered as prose

### Symptom

YAML files opened with variable-pitch fonts and visual-line wrapping, despite
the earlier change to re-fire `prog-mode-hook` (commits `0928969`, `bb70c5c`).

### Root cause

`yaml-mode` / `yaml-ts-mode` derive from `text-mode`, so `text-mode-hook` fires
before the yaml-specific hook. In `init-writing.el` that hook enables
`variable-pitch-mode`, `visual-line-mode`, and `visual-wrap-prefix-mode`, and
`jinx-mode` is also attached to `text-mode-hook` for prose spell-check. Running
`prog-mode-hook` on top added the prog niceties but did not undo the prose
ones, so the buffer kept its text-mode appearance.

### Fix

`lisp/init-lang-data.el` — extended `jotain-lang-data--enable-prog-mode-features`
to disable `variable-pitch-mode`, `visual-line-mode`,
`visual-wrap-prefix-mode`, and `jinx-mode` after running `prog-mode-hook`. The
two visual-wrap / jinx calls are guarded so the function stays safe if those
packages aren't loaded yet.
