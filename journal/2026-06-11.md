# 2026-06-11

## Upstream survey + realign `emacs.nix` (nixpkgs `emacs31`, cached overlay variants, Darwin patch cleanup)

### Summary

Surveyed upstream nixpkgs, nix-community/emacs-overlay, and
nix-giant/nix-darwin-emacs (the "nix-darwin emacs overlay" we fetch macOS
patches from) against our packaging. Several upstream moves in the last week
invalidated assumptions baked into `emacs.nix`:

- **nixpkgs ships `emacs31` again** (31.0.90 prerelease, added 2026-06-05,
  Hydra-cached, plus `emacs31-gtk3/-nox/-pgtk`). Our 2026-06-05 journal
  rationale for basing mainline on the overlay's `emacs-git` ("emacs31
  attribute no longer exists in nixpkgs") went stale within the week.
- **emacs-overlay ships prebuilt `emacs-unstable` and `emacs-igc`** (and pgtk
  siblings), all cached on nix-community.cachix.org with daily-updated rev
  pins. Our `unstable`/`igc` variants instead overrode `emacs-git`'s `src`
  with `lib.fakeHash`, forcing the `--argstr hash` dance and a from-source
  build every time.
- **nix-darwin-emacs dropped `fix-window-role.patch` from `patches-31/`**
  (fixed upstream in Emacs 31) and added `adjust-ns-init-colors.patch` to
  `patches-unstable/`. Our `withFixWindowRolePatch` was doubly dead: its
  `patchBranch == "30"` gate could never be true, and the file it pointed at
  no longer exists. Separately, macport (Emacs 30.x) mapped to `patches-31`
  — the wrong branch.
- **nixpkgs `emacs30-macport`** now builds from the jdtsmith/emacs-mac fork
  (Mitsuharu's original is dormant) and does **not** override `srcRepo`, so
  the make-emacs default (`true`) applies — our `srcRepo = false` for
  macport silently busted its cache.
- **`withXwidgets` default divergence**: our extra `isDarwin` clause was a
  leftover mirror of the overlay's `withXwidgets = false` override; with
  mainline on nixpkgs `emacs31` it would have broken parity for
  `withPgtk = true` builds vs `emacs31-pgtk`.

### Changes

- **`emacs.nix`**
  - Base package map: `mainline` → nixpkgs `emacs31`; `git`/`unstable`/`igc`
    → the overlay's `emacs-git`/`emacs-unstable`/`emacs-igc`; `macport` →
    nixpkgs `emacs-macport` alias (emacs-overlay ships no macport).
  - Source overrides only apply with an explicit `--argstr rev`; default
    revisions ride the overlay/nixpkgs pins. Dropped the manual
    `gitMeta`/fakeHash plumbing, the duplicate aarch64
    `--enable-check-lisp-object-type`, and the manual igc `--with-mps=yes` +
    `mps` buildInput (`pkgs.emacs-igc` already carries them, and
    `makeOverridable` replays its `overrideAttrs` layers through
    `.override`).
  - `srcRepo ? true` for all variants (matches upstream; every base is a git
    checkout now that nixpkgs fetches via fetchgit/fetchFromGitHub).
  - `withXwidgets` default now matches upstream make-emacs.nix:
    `!noGui && (withGTK3 || withPgtk || withNS || variant == "macport")`.
  - Darwin patches: removed `withFixWindowRolePatch`; added
    `withAdjustNsInitColorsPatch` (patches-unstable only, i.e. git/igc);
    `patchBranch` now maps macport → `"30"`; replaced every `lib.fakeHash`
    with pinned per-branch fetchpatch hashes (31 and unstable currently
    carry byte-identical patch content; 30 differs).
- **`CLAUDE.md`** — cache-parity section retargeted to `pkgs.emacs31` (+
  per-variant overlay parity checks in the verify snippet); base-package map
  and the "fails on first run, reports hash" wording corrected.
- **`Justfile`** — `build-git` comment no longer claims a hash dance.
- **`flake.nix`** — emacs-overlay input comment: the overlay now only
  supplies the git-based variants; mainline comes from nixpkgs.
- **`docs/`** (`introduction.mdx`, `installation.mdx`,
  `architecture/overview.mdx`, `architecture/nix-build.mdx`) — parity
  target, variant table, IGC/Git-variant sections, and hash-dance notes
  updated to match the new base-package map.

### Verification

- Parity eval is `true` for **all four** default-rev variants:
  `mainline == pkgs.emacs31`, `git == pkgs.emacs-git`,
  `unstable == pkgs.emacs-unstable`, `igc == pkgs.emacs-igc`.
- `nix-build emacs.nix --dry-run` → "will be fetched (75.1 MiB)", i.e. a
  pure Hydra substitution; no compilation.
- Darwin patch / macport / custom-rev paths instantiate cleanly
  (`nix-instantiate` with `system = "aarch64-darwin"` and a dummy rev).
- `nixfmt` clean; `nix flake check` green.

### Watch items (no action yet)

- **`__structuredAttrs` enabled** for Emacs on nixpkgs master 2026-06-08 —
  not in our pin (`8c3cede7`); lands at the next `just update`. Our
  `overrideAttrs` list/string manipulation is compatible, but re-run the
  parity eval after the bump. `strictDeps = true` is already at our pin.
- **homebrew-emacs-plus** has a `fix-ns-x-colors.patch` for Emacs 31 that
  nix-darwin-emacs has *not* adopted; their `adjust-ns-init-colors.patch`
  (unstable branch only) is the alternative approach to the same
  headless-dump color problem.
- nix-darwin-emacs has no `patches-32/` directory yet, while
  homebrew-emacs-plus added emacs-32 targets in May 2026 — if Emacs 32
  patch branches appear, `patchBranch` may need a new mapping.
- The mainline base is now the 31.0.90 **prerelease**; nixpkgs will bump it
  through the 31.x release cycle, with each bump arriving via `just update`
  as a cache hit.
