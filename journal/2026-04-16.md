# 2026-04-16

## Fix nix flake check

### Summary

Made `nix flake check` pass. Fixed two categories of issues:

### 1. Nixfmt formatting (`nix/use-package.nix`)

Reformatted the `if/else` expression in `toEmacsPackage` to match nixfmt's
expected style (indented `else` branch with parenthesised body).

### 2. Elisp byte-compile errors (`checks.x86_64-linux.elisp-compile`)

The check runs `batch-byte-compile` with `byte-compile-error-on-warn t`, turning
all byte-compile warnings into hard errors.

**Free variables in `early-init.el`** — Added `defvar` declarations for variables
that are only set inside conditional guards but not defined in the current
compilation context: `package-quickstart`, `warning-suppress-log-types`,
`use-package-enable-imenu-support`, `use-package-always-ensure`,
`ns-use-thin-smoothing` (macOS-only), `native-comp-async-report-warnings-errors`,
`native-comp-speed`, and `treesit-extra-load-path`.

**Sandbox `make-directory` failure** — `init-core.el` called `(make-directory
jotain-var-dir t)` at top level, which runs during batch compilation in the Nix
sandbox where `~/.emacs.d/` can't be created. Wrapped in `ignore-errors`.

**`rxt--re-builder-switch-pcre-mode` from pcre2el** — `magit-todos` loads
`pcre2el`, which has a `defadvice ... compile` that byte-compiles its advice body
at activation time. With `byte-compile-error-on-warn t` active, the forward
reference to `rxt--re-builder-switch-pcre-mode` in that advice body fails.
Fixed by adding `(require 'pcre2el)` to `nix/checks.nix` before setting the
flag, so pcre2el's advice compiles without the flag active.

**Functions not known to the byte-compiler** — Added `:functions (...)` to
`use-package` blocks in `init-ai`, `init-completion`, `init-core`, `init-editing`,
`init-navigation`, `init-org`, `init-prog`, `init-project`, `init-systems`,
`init-tracking`, `init-ui`, `init-vc`. Also added a `declare-function` for
`jotain-core--auto-create-missing-dirs` (defined inside a `:config` block).

**`diff-hl-magit-pre-refresh` obsolete** — Replaced with `ignore` as the
diff-hl 1.11.0 changelog requires.

**`treesit-fold-indicators-priority` free variable** — Changed `:init (setq ...)
` to `:custom (...)` so the variable is set via `customize-set-variable` after
the package loads (when the variable is bound).
