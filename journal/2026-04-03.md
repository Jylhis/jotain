# 2026-04-03

## PostHog Telemetry Integration

Added `jotain-telemetry.el` — a PostHog analytics integration to track Emacs usage patterns.

### What was done

- Created `jotain-telemetry.el` with a global minor mode (`jotain-telemetry-mode`)
- Wired it into `init.el` via `use-package`
- Targets PostHog EU Cloud (`eu.i.posthog.com`)

### Events tracked

1. **session_start/session_end** — session duration, Emacs version, system info
2. **major_mode_time** — time spent in each major mode (periodic)
3. **command_frequency** — top 20 most-used commands per flush interval
4. **packages_loaded** — all loaded packages (reported once after init)
5. **error_occurred** — unhandled errors with debouncing (max 1 per 10s)

### Design decisions

- **Zero external dependencies** — uses built-in `url-retrieve` and `json-serialize` (Emacs 30+)
- **Batched events** — events queue in memory and flush every 5 minutes or on exit
- **Async HTTP** — `url-retrieve` for periodic flushes, `url-retrieve-synchronously` on kill-emacs
- **Privacy** — persistent anonymous UUID stored in `~/.config/jotain/posthog-id`; API key via `POSTHOG_API_KEY` env var (or defcustom fallback); telemetry silently disabled if no key
- **Performance** — `post-command-hook` only does a hash-table increment; mode hook just records a timestamp
- **Opt-out** — set `jotain-telemetry-enabled` to nil or simply don't set an API key

## Telemetry Refactor & Bug Fixes

### What was done

- Moved `jotain-telemetry.el` to `lisp/` directory and added `lisp/` to `load-path` in `init.el`
- Added `jotain-telemetry-debug` option — when enabled, logs all events to `*jotain-telemetry*` buffer with timestamps (capped at 100k chars to prevent unbounded growth)
- Fixed `[object Object]` bug in PostHog — `command_frequency` commands were sent as an array of `{command, count}` objects which PostHog stringified poorly; now sent as a flat object mapping command names to counts
- Extracted duplicated HTTP flush logic into `jotain-telemetry--send-batch` with a `synchronous` parameter — both `flush` and `--session-end` now call it
- Fixed `--get-distinct-id` reading the ID file twice — now reads once and reuses
- Fixed `buffer-list-update-hook` overcounting — it fires on every buffer list change (switching, reordering), not just creation; now tracks actual buffer count delta
- Fixed stray `9u` typo in `init.el` header
