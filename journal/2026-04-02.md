## Tree-sitter grammars

Split the Nix build into two files:
- `emacs.nix` — bare Emacs build from source (renamed from `default.nix`)
- `default.nix` — distribution layer that wraps Emacs with all tree-sitter grammars

Used the NixOS wiki-recommended approach: `emacsPackagesFor`/`withPackages` with
`epkgs.treesit-grammars.with-grammars`. This installs all 275 tree-sitter grammars
from nixpkgs and auto-generates a `site-start.el` that sets `treesit-extra-load-path`,
so no changes to `init.el` are needed.

## Added ellsp and Elsa to devenv-emacs-lisp.nix

Researched upstream devenv language modules (nix.nix, clojure.nix, javascript.nix) from
`/nix/store/qvzvk0z65dx8vf33zfa87bnkd9viq16h-source/src/modules/languages/` to understand the
standard LSP option pattern: `lsp.enable` (default true) + `lsp.package`.

### ellsp — Emacs Lisp Language Server

- NOT in nixpkgs (not even under emacsPackages)
- Ships as pre-built platform-specific binaries on GitHub releases (Node.js compiled via `pkg`)
- Created `nix/ellsp.nix`: fetches the correct binary for the platform (linux-x64, linux-arm64,
  macos-x64, macos-arm64) from GitHub releases v0.1.0
- Uses `autoPatchelfHook` on Linux for dynamic linking
- Note: the v0.1.0 binary has a broken Node.js embedded runtime (corrupted `pkg/prelude/bootstrap.js`).
  The packaging is correct but the upstream binary is broken. Will need a fixed upstream release.

### Elsa — Emacs Lisp Static Analyzer

- Available as `pkgs.emacsPackages.elsa` in nixpkgs (v20250717.1205)
- But it's an Emacs Lisp package (`.el` files), not a standalone binary
- Created `nix/elsa.nix`: wraps Emacs with Elsa loaded via `emacsPackagesFor`/`withPackages`,
  then creates a shell script that invokes `emacs --batch --load=elsa --funcall=elsa-run`
- This mirrors the upstream `bin/elsa` wrapper pattern

### devenv module changes

Updated `nix/devenv-emacs-lisp.nix` following upstream devenv patterns:
- Added `lsp.enable` / `lsp.package` options (defaults to ellsp binary)
- Added `elsa.enable` / `elsa.package` options (defaults to wrapped Elsa)
- Both default to `enable = true` following the upstream convention
- Uses `lib.optional cfg.lsp.enable cfg.lsp.package` pattern from nix.nix/clojure.nix

### Debugging note

After updating the module, `devenv shell` didn't show the new packages because of a stale
`nix-eval-cache.db`. Deleting `.devenv/nix-eval-cache.db*` and re-running fixed it.
Both `ellsp` and `elsa` then appeared in `$DEVENV_PROFILE/bin/`.
