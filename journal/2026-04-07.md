# 2026-04-07

## Editor instrumentation for activity tracking

Wired Emacs into a local-first activity tracking pipeline (Wakapi for
coding heartbeats, ActivityWatch for window/idle, keyfreq for command
frequency, org-clock for task time) and disabled the PostHog telemetry
module.

### Added to `init.el`

A new "Activity tracking" section with five `use-package` forms:

- **`wakatime-mode`** — heartbeats to a self-hosted Wakapi instance.
  Guarded with `:if (executable-find "wakatime-cli")` so the config
  still loads cleanly on hosts without the CLI. Configuration of
  `~/.wakatime.cfg` (api_url, api_key) is a host-side concern, not
  jotain's.
- **`activity-watch-mode`** — heartbeats to ActivityWatch. Activated
  unconditionally; the package degrades gracefully when the
  `aw-server` isn't reachable, so it's safe even on hosts that don't
  run the daemon.
- **`keyfreq`** — command frequency tracking, with the persistent
  autosave mode enabled so stats survive restarts. Data lands in
  `~/.emacs.keyfreq` and is consumable by external analysis tools.
- **`org-clock`** — built-in, but now wired with persistence
  (`org-clock-persist`, `org-clock-idle-time 15`,
  `org-clock-into-drawer`) and `org-clock-persistence-insinuate`.
  Loaded eagerly (no `:defer t`) so the persistence hooks attach at
  startup — otherwise a clock that was running in the previous
  session won't be restored to the mode line until something else
  triggers org-clock. First pass had `:defer t` here and got this
  wrong.
- **`org-clock-csv`** — exporter, deferred via `:defer t` so the
  package isn't loaded eagerly but its commands (e.g.
  `org-clock-csv-export`) remain autoloaded and discoverable from
  `M-x`. First pass used `:after org-clock`, which blocks autoload
  registration until org-clock is loaded and hides the commands from
  `M-x` discovery; `:defer t` achieves the same "don't load eagerly"
  goal while preserving autoloads.

### Disabled `jotain-telemetry`

`jotain-telemetry-mode` (PostHog batch upload of command counts, mode
times, session events, errors) is now turned off via
`(jotain-telemetry-enabled nil)` and the explicit `(jotain-telemetry-mode 1)`
call removed. The module file stays in `lisp/` for now — only the
init-time activation is gone. The new tracking stack covers the same
ground locally, without shipping events to a third party.

### Design choices

- **`:custom` over `setq`**: enforced by the `elisp-conventions`
  skill — `setopt`/`:custom` runs the defcustom `:set` function and
  validates `:type`, neither of which `setq` does. So org-clock
  customizations go through `:custom`.
- **Conditional `wakatime-mode`**: activating `global-wakatime-mode`
  unconditionally throws errors at startup on machines without
  `wakatime-cli`. `executable-find` is the feature-gate; this matches
  jotain's preference for `fboundp`/runtime detection over hard
  version checks.
- **No new `lisp/jotain-tracking.el` module**: the wiring is purely
  declarative `use-package` forms with no custom logic, so it lives in
  `init.el` alongside `nix-ts-mode`, `markdown-mode`, and `magit`.
  `jotain-telemetry.el` exists as its own module because it implements
  custom event-collection logic; the activity-tracking packages are
  off-the-shelf, so a wrapper module would be ceremony with no payoff.

## `jotain-performance` module — performance tweaks

Added `lisp/jotain-performance.el` collecting four performance
adjustments lifted from Doom Emacs (and seen in Purcell/Centaur for
the LSP one), via
<https://emacsredux.com/blog/2026/04/07/stealing-from-the-best-emacs-configs/>.

- **Disable bidirectional text scanning** —
  `bidi-display-reordering`/`bidi-paragraph-direction` forced to
  `left-to-right` via `setq-default` (both are buffer-local), and
  `bidi-inhibit-bpa` enabled via `setopt`.  Skips Emacs's
  bidirectional parenthesis algorithm on every redisplay; cost is
  negligible in small buffers but compounds in large JSON/log files.
  Safe because I don't edit RTL scripts.
- **Skip fontification during input** — `setopt
  redisplay-skip-fontification-on-input t` defers
  syntax-highlighting work while keys are coming in.  Removes
  micro-stutters in tree-sitter modes; highlight catches up the
  instant input pauses.
- **4 MB process output buffer for LSP** — `setopt
  read-process-output-max (* 4 1024 1024)`.  Default 64 KB is way
  too small for `rust-analyzer`/`clangd` payloads; cuts read-call
  count.  Free perf for `eglot`.
- **No phantom cursors / non-selected highlights** — `setq-default
  cursor-in-non-selected-windows nil` (buffer-local) and `setopt
  highlight-nonselected-windows nil`.  Mostly visual but trims a
  little rendering work in multi-window frames.

Wired into `init.el` via a bare `(use-package jotain-performance)`
form with no customization — every setting is unconditional and
side-effect-free at top-level.

### Style notes

- `setopt` for everything declared as `defcustom` (validates `:type`,
  runs `:set`); `setq-default` for `bidi-display-reordering` and
  `cursor-in-non-selected-windows` because they're buffer-local
  defaults rather than ordinary user options.
- All four C-level vars (`bidi-inhibit-bpa`,
  `redisplay-skip-fontification-on-input`, `read-process-output-max`,
  `highlight-nonselected-windows`) are preloaded, so no
  `with-eval-after-load` guard is needed.

## `jotain-performance` module — ergonomic hacks

Extended `jotain-performance.el` with a second section of small
ergonomic adjustments from the same emacsredux roundup.  These are
not strictly performance tweaks but they fit in the same
"unconditional QoL" bucket and don't deserve a module of their own.

- **`save-interprogram-paste-before-kill t`** — clipboard contents
  copied from another app are pushed onto the kill ring before a
  kill replaces them, so `M-y` still walks back to the URL you just
  copied from the browser.
- **`kill-do-not-save-duplicates t`** — consecutive identical kills
  no longer eat slots in the ring.
- **Auto-chmod scripts on save** — `after-save-hook` runs
  `executable-make-buffer-file-executable-if-script-p`, so any file
  with a `#!` shebang becomes executable on first save.  Built-in
  function, no extra package needed.
- **`ffap-machine-p-known 'reject`** — stops `find-file-at-point`
  from doing live network reachability checks on hostnames it
  spots under point.  Wrapped in `with-eval-after-load 'ffap`
  because the defcustom doesn't exist until `ffap` loads.
- **`window-combination-resize t`** — splits resize all siblings
  proportionally instead of always halving the selected window.
- **`set-mark-command-repeat-pop t`** — after one `C-u C-SPC`,
  plain `C-SPC` keeps popping the mark ring.  Makes the mark ring
  actually pleasant to navigate.
- **Recenter after `save-place-mode`** — defadvice (well,
  `advice-add :after`) on `save-place-find-file-hook` calls
  `recenter` so reopening a file doesn't leave point on the
  bottom line of the window.  Wrapped in
  `with-eval-after-load 'saveplace`.
- **Reversible `C-x 1`** — `winner-mode` is enabled, and
  `C-x 1` is rebound to `jotain-performance-toggle-delete-other-windows`,
  which collapses to a single window if there are several visible,
  or calls `winner-undo` if the frame is already a single window
  and a previous configuration is available.

### Style notes

- Hook function and advice function are named (`jotain-performance--recenter-after-save-place`,
  the toggle command), never lambdas — required by the
  `elisp-conventions` skill so they're removable, debuggable, and
  don't double-register on file reload.
- `keymap-global-set` instead of legacy `global-set-key`.
- `with-eval-after-load` guards for vars/hooks that live in
  not-yet-loaded files (`ffap`, `saveplace`).
