# Architecture Overview

Jotain is organized into distinct layers, each with a clear responsibility.

## File Structure

```
jotain/
├── early-init.el      # Pre-package/GUI initialization (Emacs 30+)
├── init.el            # Main configuration entry point
├── lisp/              # Custom Emacs Lisp modules
├── default.nix        # Distribution layer (Emacs + tree-sitter grammars)
├── emacs.nix          # Emacs build expression
├── nix/               # Additional Nix expressions
├── scripts/           # Development scripts (linting, evaluation)
├── test/              # ERT test files
├── devenv.nix         # Development environment
├── Justfile           # Task runner commands
├── docs/              # Reference documentation
└── journal/           # Development journal entries
```

## Layers

### Emacs Lisp Layer

The Elisp configuration is split into three parts:

1. **`early-init.el`** — Loaded before package initialization and GUI setup. Handles settings that must be applied before the UI renders, such as disabling the tool bar and startup screen.

2. **`init.el`** — Main configuration entry point. Sets up package archives, loads the theme, configures base Emacs settings, and loads custom modules from `lisp/`.

3. **`lisp/`** — Custom modules following the `jotain-*` naming convention. Each module is a self-contained feature.

### Nix Layer

The Nix layer provides reproducible Emacs builds:

1. **`emacs.nix`** — Comprehensive build expression supporting multiple source variants, GUI toolkits, native compilation, and tree-sitter.

2. **`default.nix`** — Distribution wrapper that imports `emacs.nix` and adds tree-sitter grammars via `emacsPackagesFor`/`withPackages`.

### Development Layer

- **`devenv.nix`** — Development shell with tooling
- **`Justfile`** — Convenience commands for building, running, and cleaning
- **`scripts/`** — Linting and evaluation scripts
- **`test/`** — ERT (Emacs Lisp Regression Testing) test files
